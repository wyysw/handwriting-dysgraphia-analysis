#### 三、从特征到分类诊断的组织方法（三种方案并列，对照实验）

你自己在 README 里说希望"相对正常样本参考分布计算偏离度，双轴组合映射等级"。我把它落到具体可实现的三种方案，**都做**，互为对照：

##### 方案 A：固定权重打分法（README 模块2风格，你要求必做的 baseline）

$$\text{score} = \sum_i w_i \cdot (1 - \text{norm}(f_i))$$

- 权重 $w_i$ 按照规则文档给定（例如 1A 权重 0.3、1B 0.3、1D 0.3、3 占 0.1）
- 阈值法分级
- **优点**：完全可解释、零数据需求
- **缺点**：权重拍脑袋、不能跨游戏、没有学习

##### 方案 B：双轴偏离度法（你研究的主角）

分三步：

**Step 1 — 参考分布建立**：仅在 **normal 样本**上，对每个特征 $f_i$ 算均值 $\mu_i$、标准差 $\sigma_i$（样本少就用 MAD 代替 σ 更稳健）。

**Step 2 — 偏离度计算**：对任意样本，每个特征的偏离度

$$d_i = \frac{|f_i - \mu_i|}{\sigma_i + \epsilon}$$

（对于"越大越好"的特征取单边 ReLU）

**Step 3 — 轴内聚合**：

$$F\_\text{score} = \sqrt{\frac{1}{|F|}\sum_{i\in F} d_i^2}, \quad C\_\text{score} = \sqrt{\frac{1}{|C|}\sum_{i\in C} d_i^2}$$

用 L2 范数而不是均值的原因：更能捕捉"某一维严重异常"的情况，对书写障碍这种"局部性缺陷"更敏感。

**Step 4 — 分级映射**：在 $(F, C)$ 二维平面上划分：
- $F$ 低 & $C$ 低 → 正常
- $F$ 高 & $C$ 低 → 功能受损型（可能是注意力/理解问题）
- $F$ 低 & $C$ 高 → 控制受损型（典型 dysgraphia 运动模式）
- $F$ 高 & $C$ 高 → 重度受损

**这样做的方法论贡献**：比方案 A 多了"在正常参考分布上锚定"，比直接分类器多了"可解释的双轴几何语义"。这两点都是本科毕设能说出的亮点。

##### 方案 C：轻量级有监督分类器

输入：$(F\_\text{score}, C\_\text{score})$ 两维 + 几个最重要的原始特征（比如 F1, C1）
模型选 **逻辑回归** 或 **depth=3 的决策树**——这两个在小数据上最稳，且都能画决策边界，论文里放图好看。

**不要**用 RF/XGBoost，在 <100 样本规模上它们的方差会让结果不可复现。

#### 四、后续实验要点（简述）

1. **数据侧**
   - 先确定 normal 样本数量 N，这是方案 B 能不能成立的前提。建议至少 20 个 normal 才能估 μ/σ。
   - 患病样本不需要特别多，因为你不是训练深网络，而是做对比试验。
   - 三个游戏每个样本都要完整采集（不能只有一个游戏的数据）。

2. **特征提取侧**
   - 控制轴的 C1–C3 可以直接复用你模块2里已经写好的 `analyze_line_control`，只需把"标准线段"换成迷宫的骨架线。
   - 迷宫的 F1（路径覆盖率）需要**通道内腐蚀+距离变换**，这和圆形迷宫的线条掩码正好对接，实现成本低。
   - F5 越界/触边按"深度×时长"加权，不是按次数——原文档也是这么说的，避免短暂抖出被过度惩罚。

3. **验证侧**
   - 样本少 → 用 **Leave-One-Out CV**，不要用 holdout。
   - 关键指标：准确率之外必须报 **Sensitivity (召回)** 和 **Specificity**，临床场景里漏诊成本远高于误诊。
   - **跨游戏泛化测试**：在画对称+方形迷宫训练，在圆形迷宫测试，反之同理。这是你说"分类器对三个游戏都有效"的直接证据。三种方案都要测一遍，**方案 B 的通用性应该显著优于方案 A**，这是论文核心表格。

4. **消融**
   - 仅 F 轴 / 仅 C 轴 / F+C 联合
   - 用 μ/σ 参考分布 vs 不做参考分布（直接原始特征）
   - 各主要特征去掉一个看分类下降多少（告诉读者哪个特征最关键）

5. **陷阱预警**
   - **情绪性乱画 gate** 必须先于所有打分运行，否则会把乱画样本算成"重度受损"，污染参考分布。
   - 三个游戏的**特征量纲要在同一 normal 基线上归一化**，不要每个游戏各自归一化，否则跨游戏泛化没法做。
   - 关键点/闭合阈值（如 6px、≤3px）**不要写死**，做成和 `step_x / step_y` 成比例，这就是原文档里 "跨设备可比" 那一条的技术含义。

# 回答2：

我读完 README 和 wen1.md 了。总体判断：wen1.md 的双轴+三方案对照+7维抽象这个骨架是对的，但有一处关键的方法论漏洞没处理，另外特征数量对你的数据量来说偏多。下面给我的定稿，不是逐条对比，而是给一个直接可执行的版本，最后诚实评估难度和工作量。

## 一、先补一个 wen1.md 没讲清楚的核心问题：样本粒度

你的数据是"每个孩子玩 3 个游戏"，所以**一条样本到底是什么**必须先定义。有两种选择：

- **per-game 样本**（3N 条）：同一个孩子的 3 条样本不独立，CV 会被污染，分类器实际测到的是"同一个孩子其它两游戏泄漏过来"的信息
- **per-subject 样本**（N 条）：每个孩子产生一组跨 3 游戏的特征，分类标签是 subject 级的"有无书写障碍"

你的临床诊断目标是"这个孩子有没有书写障碍"，所以**样本粒度必须是 subject**。这决定了后面所有聚合、归一化、CV 划分的做法。wen1.md 隐含假设了这一点但没点破，一旦按 per-game 写代码就会全盘出错。

具体做法：每个游戏各自算出它的 F_score、C_score 两维（即 wen1.md 方案B的产物），3 个游戏拼成 subject 的 6 维特征向量；下游分类器吃这 6 维。

## 二、特征定稿（比 wen1.md 精简，避开小样本过拟合）

14 个特征 × 3 个游戏 = 42 维原始特征，对 N<30 的数据量是灾难。我砍到 **每游戏 8 维**，理由是砍掉的都是和保留项强相关的衍生量。

**功能轴 F（每游戏 4 维）**

| ID | 名称 | 对称 | 方形迷宫 | 圆形迷宫 |
|---|---|---|---|---|
| F1 | 完成度 | 翻转后 mask 与 blue 的 tolerant-F1 | 正确路径覆盖率（通道内腐蚀后） | 同方形，通道沿环展开 |
| F2 | 关键结构对齐 | 网格交点命中率（阈值按 step_x/step_y 比例） | 岔路决策点按序经过率 | 角度采样点覆盖率 |
| F3 | 无效书写 | 图形外像素占比 + 对称轴另一侧误画 | 越界面积×深度加权 + 触边时长 | 同方形 |
| F4 | 回退/反向 | —（对称任务回退概念弱，可置 0 或合并 F3） | 走错回退路径长 / 总路径长 | 同方形 |

**控制轴 C（全游戏共用定义，这是跨游戏通用的关键）**

| ID | 名称 | 定义 |
|---|---|---|
| C1 | 抖动比例 | 沿标准线段局部坐标 d 的残差超容差点数占比 |
| C2 | 分段策略性 | 短笔段（<L_threshold）总长 / 总笔迹长 |
| C3 | 速度变异 | 速度序列 CV = σ/μ |
| C4 | 停滞密度 | 低速时长 / 落笔总时长 |

**砍掉的项和理由**：F2(size)与F1强相关；F4(闭合)在迷宫里定义不自然，可并入F1；C2(严重抖动)、C3(抖动发作数)都是C1的衍生；C5(抬笔密度)与C2强相关；C7(压力CV)Wacom采样未必稳定，保留会增加噪声。

**归一化策略（和 wen1.md 不同的一点）**：wen1.md 建议"三游戏在同一 normal 基线归一化"，但正常儿童在圆形迷宫 vs 方形迷宫上本来就有系统性差异（弧线难于直线），强行共基线会把游戏难度误判成异常。**每个游戏在它自己的 normal 样本上各算 (μ, σ)，偏离度在各游戏内归一后再跨游戏聚合**——这才是"跨游戏可比"的正确含义。

**乱画 gate 的可实现触发条件**（wen1.md 说要做但没给定义）：同时满足即触发 `is_scribble=True`，跳过打分：
- 用户 mask 与标准 mask 的 IoU < 0.05
- F3(无效书写) > 0.5  
- 笔迹总长 / 标准路径长 > 3

## 三、分类方法：三方案并列对照

- **方案 A（baseline）**：按规则文档的固定权重加权打分，阈值分级。**必做，因为这是你毕设的对照组**，证明"学习 + 参考分布"的方法论贡献。
- **方案 B（主方法）**：双轴偏离度。在 normal 样本上算 (μ, σ) 用 MAD 稳健估计，每个特征的偏离度用单边 ReLU（只惩罚"越差越偏"的方向），轴内用 L2 聚合（对单点严重异常敏感）。F、C 双轴平面上划四象限做分级。
- **方案 C（对比）**：输入 6 维 subject 特征（3游戏×2轴）+ 少数几个最重要的原始特征，用 **Logistic Regression** 或 **depth=3 决策树**。不要 RF / XGBoost——在 N<50 上方差大到结果不可复现，毕设答辩时会被质疑。

## 四、实验要点

1. **CV 策略**：**Leave-One-Subject-Out**，不是 leave-one-sample-out。
2. **指标**：Accuracy 之外必报 Sensitivity、Specificity、AUC，临床场景漏诊成本>>误诊。
3. **跨游戏泛化实验**（论文核心表）：用 2 个游戏训练、第 3 个游戏测试，3 种组合各跑一遍。这是"分类器对三个游戏都有效"的直接证据。方案 A 做不了（权重是游戏特定的），这恰好说明方案 B/C 的方法论价值。
4. **消融**：仅 F 轴 / 仅 C 轴 / F+C 联合；有无 normal 参考基线；每次去掉一个特征看下降。
5. **陷阱**：乱画 gate 必须先于一切打分跑，否则会污染 normal 参考分布；normal 样本 N<20 时方案 B 的 (μ,σ) 不可靠，这时 B 会输给 A，要在论文里诚实承认这个前提。

### 功能轴 F：**孩子有没有完成任务目标**

这一轴关心的是宏观结果——画得像不像目标图形。不关心过程。

#### F1 完成度（Completion）

**直觉**：画够了吗？书写障碍儿童可能画一半就不画了，或者只画出模糊轮廓。

**三个游戏的定义**：
- **对称游戏**：把蓝图形沿对称轴翻转得到"期望图形"，计算用户笔迹与期望图形的**容差 F1**——就是允许小幅错位的情况下，用户画的像素和期望像素的重叠程度（0 到 1）。容差具体做法：把期望图形做 2–3 像素的膨胀（向外扩一点）再对比，避免小偏移被严判。
- **方形迷宫**：取"正确路径通道"（通道本来就有一定宽度），看用户笔迹在通道内的像素数占通道总像素数的比例。
- **圆形迷宫**：同方形迷宫，但通道是环形的。

**输出**：0 到 1，越大越好。

#### F2 关键结构对齐（Key Structure Alignment）

**直觉**：孩子画出的东西"骨头对不对"？比如对称图形有几个关键交点，迷宫有几个必经的转弯点，这些关键位置有没有被经过。

**三个游戏的定义**：
- **对称游戏**：找蓝图形骨架上的关键点（网格交点、端点、分叉点），在用户笔迹翻转后的对应位置附近（允许一定容差）找有没有笔迹覆盖。命中率 = 命中的关键点数 / 总关键点数。
- **方形迷宫**：从正确路径骨架上找**决策点**（岔路口）和**转角**，检查用户笔迹按顺序经过了几个。
- **圆形迷宫**：在环形路径上每隔一定角度（比如 15°）采一个点，看覆盖率。

**输出**：0 到 1，越大越好。

**术语说明**：
- **骨架点**：前面说过的"骨架线"上的点。对骨架求"分叉点"就是有三条或以上线条在此汇合的点。
- **容差**：允许误差半径，比如 5 像素。

#### F3 无效书写（Invalid Drawing）

**直觉**：孩子画了多少不该画的东西？书写障碍的表现之一是在目标图形以外乱画、越过边界。

**定义**（三个游戏共用）：

$$\text{F3} = \sum_{p \in \text{用户笔迹}} \mathbb{1}[p \notin \text{合法区域}] \cdot d(p)$$

其中 $d(p)$ 是这个违规像素到合法区域边界的最短距离（**距离变换**给出）。

**解释**：
- 对称游戏的"合法区域"= 对称轴某一侧的整个画布（越过对称轴算违规，在对称的同一侧乱画也算违规、用"离蓝图形距离 × 像素数"衡量）
- 迷宫的"合法区域"= 通道内部
- **距离变换**：对一张二值图算"每个像素到最近的 1 像素的距离"的标准图像处理操作，OpenCV 一行代码 `cv2.distanceTransform`
- 越界得越深、面积越大，F3 越大

**输出**：正数，越小越好。最后除以"标准图形面积"做尺度归一。

#### F4 路径偏离比（Off-Path Ratio）

**直觉**：从"画图过程"的角度补充 F1——F1 只看结果有没有覆盖，F4 看有多少笔迹走到了不该走的地方。比 F3 粗一些，但对迷宫特别适用（走错支路）。

**定义**：

$$\text{F4} = \frac{\text{用户笔迹中位于正确通道外的像素数}}{\text{用户笔迹总像素数}}$$

**和 F3 的区别**：F3 用"深度 × 面积"强调"错得多离谱"；F4 只算"错的比例"，更鲁棒，不会被偶尔的大幅跨界支配。两个信号互补。

**对称游戏的适配**：把"正确通道"替换为"期望图形的膨胀版本"（即把翻转后的蓝图形向外扩几像素当作"正确区域"）。

**输出**：0 到 1，越小越好。

### 控制轴 C：**孩子手的精细控制能力**

这一轴关心过程——即使最终画出来了，过程里有没有抖、有没有反复停停走走。这是书写障碍儿童和正常儿童最有判别力的一轴。

#### C1 抖动比例（Jitter Ratio）

**直觉**：画应该平滑的线时，有没有小幅颤抖。书写障碍儿童画直线常常是"一小段一小段拐"的。

**定义（通用做法）**：
1. 对每一笔（两次抬笔之间的连续轨迹），找这笔应该对应的"理想参考线"——
   - 对称游戏：翻转后蓝图形上最近的骨架段
   - 迷宫：正确通道的骨架段
2. 把笔迹点投影到参考线上，算每个点到参考线的**垂直距离**（残差）
3. 设一个容差阈值 $\tau$（比如 3 像素），残差超 $\tau$ 的点数占这笔总点数的比例

最后取所有笔的加权平均（按笔长加权）。

**输出**：0 到 1，越小越好。

**术语说明**：
- **投影到参考线**：笔迹点 P，在参考线上找离 P 最近的点 P'，距离 |PP'| 就是残差。

#### C2 短笔段比例（Short-Stroke Ratio）

**直觉**：正常儿童倾向于一笔画完一段，书写障碍儿童倾向于"描一下、停、再描一下"，导致大量短小的笔段。

**定义**：
1. 按 pressure=0 切分原始数据，得到若干"笔段"
2. 设短笔阈值 $L_{short}$（经验值：画布对角线长度的 2%，或按正常样本分布取 25% 分位数）
3. C2 = 短于 $L_{short}$ 的笔段的总长度 / 所有笔段总长度

**注意**：分母用"总长度"而不是"笔段数"，这样一个孩子画了一条长正线 + 几根短抖动线时，C2 反映的是"短段占比"而不是被抖动次数直接支配。

**输出**：0 到 1，越小越好。

#### C3 压力变异系数（Pressure CV）

**直觉**：书写障碍儿童常常压力控制不稳，要么忽重忽轻，要么一直用力过猛。

**定义**：

$$\text{C3} = \frac{\sigma(\text{pressure})}{\mu(\text{pressure})}$$

只对 pressure > 0 的点算。σ 是标准差，μ 是均值。这个比值称为**变异系数**（CV），它的好处是消掉了压力绝对值（不同孩子用笔力道基线本来就不同）只看波动程度。

**输出**：正数，越接近 0 越稳，越大越不稳。

**一个警告**：Wacom 设备在笔尖刚落下和抬起瞬间压力采样抖动较大，建议每一笔段掐头去尾 2–3 个点再算。用 1.txt 先看一下压力分布，如果前后端确实有尖刺，就做这个处理。

## 三、`maze_geometry.py` 的圆形分支详细设计

### 3.1 顶层接口

```python
def build_maze_geometry(mask_path, game_type='maze', **kwargs):

    if game_type == 'maze':

    elif game_type == 'circle':

```

两个分支都返回同样结构的 dict：
```
{
  canvas_mask, wall_mask, channel_mask, channel_skeleton, channel_dist,
  entry_xy, exit_xy,
  solution_polyline, solution_skeleton_mask, solution_channel_mask,
  meta: {...}
}
```

### 3.2 `_build_circle_geometry` 七步流程

**步骤 1 —— 拟合外边界圆（替代方形的"floodFill 外框填充"）**

方形迷宫的外框是封闭矩形，可以直接 floodFill。但圆形迷宫外环**有两个缺口**（入口/出口），floodFill 会从缺口漏入内部，不能直接用。

改用**圆拟合**：
```python
ys, xs = np.where(wall_mask > 0)
# 外环像素 ≈ 距质心最远的那圈
r_all = np.sqrt((xs-cx_bbox)**2 + (ys-cy_bbox)**2)
outer_pixels_mask = r_all > np.percentile(r_all, 90)  # 取最外 10% 像素
# 用这些像素最小二乘拟合 (cx, cy, r_out)
cx, cy, r_out = fit_circle_LSQ(xs[outer_pixels_mask], ys[outer_pixels_mask])

canvas_mask = 填充圆盘(center=(cx,cy), radius=r_out + 2)  # +2 确保含墙
```

参数建议：`r_out ≈ 415 px`，`(cx, cy) ≈ (598, 812)`（从我的分析得出，可以作为硬编码 fallback）。

**步骤 2 —— 通道掩码（公式同方形）**

```python
channel_mask_raw = canvas_mask & ~dilate(wall_mask, r_wall=2)
```

**步骤 3 —— 【圆形新增】连通域过滤，去除中心"伪通道"**

```python
# 中心装饰小圆围出的封闭空间会成为孤立 channel blob
components = connectedComponents(channel_mask_raw)
channel_mask = 仅保留最大连通域（主环系统）
```

这一步在方形迷宫里不需要；在圆形里必须加。

**步骤 4 —— 骨架与距离变换**（与方形完全相同）

```python
channel_skeleton = skimage.morphology.skeletonize(channel_mask)
channel_dist = cv2.distanceTransform((1-channel_mask).astype(uint8), DIST_L2)
```

与方形的区别仅在于：**圆形骨架含有环状拓扑（同心环本身就是 cycle）**。不剪枝，保留完整骨架。

**步骤 5 —— 入口/出口检测（替代方形的"四边连通域"法）**

```python
# 在外环内侧 2 px 处做角度扫描
r_scan = r_out - 2
for θ in [0, 2π) step 1°:
    p = (cx + r_scan·cos θ, cy + r_scan·sin θ)
    record channel_mask[p]
# "连续 True"的两个角度段就是两个缺口，取每个缺口的质心作为 entry/exit 锚点
```

Fallback：函数签名接受 `entry_xy`/`exit_xy` 参数手动指定，与方形对齐。

**具体的硬编码建议值**（为本迷宫）：
- `entry_xy ≈ (292, 520)`（θ ≈ -135°, r ≈ r_out），即左上缺口中心
- `exit_xy ≈ (895, 1096)`（θ ≈ 45°, r ≈ r_out），即右下缺口中心

**步骤 6 —— 骨架最短路（算法同方形，BFS/Dijkstra）**

```python
entry_anchor = 骨架上距 entry_xy 最近的像素
exit_anchor  = 骨架上距 exit_xy 最近的像素
path_pixels = shortest_path_on_skeleton_graph(entry_anchor, exit_anchor)
```

**⚠️ 圆形的拓扑特殊性**：同心环会在骨架图里形成 cycle，意味着在某些局部段上可能存在"顺时针 vs 逆时针"两条等长或近等长的合法路径。BFS 只会返回其中一条。我在"设计决策"里把这个列出来讨论。

**步骤 7 —— 三种解路径表示**（与方形完全相同）

```python
solution_polyline      = path_pixels 有序列表（可 Douglas-Peucker 简化，注意保留曲线）
solution_skeleton_mask = 1px 渲染
solution_channel_mask  = dilate(solution_skeleton_mask, r=28)
```

**关键：DP 简化不要用在 polyline 上**，或者要把 ε 设得非常小（≤1 px），否则圆弧会被粗化成折线、影响 F2 沿弧长采样的密度。**建议圆形分支跳过 DP 简化**，直接用原始骨架像素序列（1201×1601 画布、几千个点无性能问题）。

| F4 | `\|user ∩ ¬ch\|/\|user\|` | 同左 | **否** |

### 4.2 C1–C3：C2/C3 复用，C1 需要讨论

**C2, C3**：直接复用 `stroke_utils.compute_C2_short_stroke_ratio` 和 `compute_C3_pressure_cv`，游戏无关。

**C1**：**这里存在阶段 2 设计与特征定义总表的不一致，需要你决策**。

特征总表（文档第 328 行）定义 C1 为"笔迹到参考骨架的残差越界比例（三游戏统一）"——即基于 `channel_skeleton` 的距离残差。

但阶段 2 的实际实现换成了"Hough on user mask"——即在用户笔迹上检测 Hough 直线段、测点到这些直线段的残差。理由是方形迷宫的理想路径是直线段。

**问题**：圆形迷宫的理想路径是**弧线+放射状直线**。Hough 只能检测直线，对弧线段无能为力。如果照搬阶段 2 的 Hough-on-user 方案，C1 会只评估用户走直线部分的抖动，系统性漏掉占比更大的弧线部分。

**我的建议（方案 A，推荐）**：圆形迷宫的 C1 回归到特征总表的统一定义——**用户笔迹点到 `channel_skeleton` 的距离残差**：

```python
for 用户笔迹的每个点 p:
    找 channel_skeleton 上最近的像素 q
    residual = distance(p, q)
    if residual <= channel_half_width (=28):       # 点在通道内，有效投影
        if residual > jitter_tol (=3):
            bad += 1
        total += 1
C1 = bad / total
```

这个公式对直线和曲线**都适用**，也是论文里"三游戏统一"最干净的表述。

**方案 B（可选）**：保留 Hough 方案，但改为在 `channel_skeleton` 上做 Hough（参考），而不是在 user mask 上。对圆形几乎检测不到线——不推荐。

**方案 C（可选）**：圆形用方案 A 的骨架距离法，方形同时用骨架距离法（顺手统一，代价是方形 C1 要重测）。如果你愿意重跑方形已有结果，这是最干净的。
